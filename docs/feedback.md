# フィードバック記録

## フィードバック履歴

### 2024-02-15 - 初期フィードバック

#### フィードバック内容

> 1. 質問は4つの軸に応じて4ブロックに分ける。各ブロックごとに7～8問ほど質問がある。このブロックごとにページを遷移させるため、1つの質問は、「質問テキスト、尺度の説明用テキスト、6つの尺度による丸いボタン」のようなMBTIと同じ形式にして。
> 2. 全体的にMBTIと同じUIにしてほしい
> 3. トーンはポップで洗練されたカラフルなデザインにしてほしい。なぜなら診断結果や4つの軸はカラフルになっているはずだから

#### 派生する問題点・考慮事項

##### 問題1: 質問数の増加
- **現在**: 16問（4軸×3問 + 互換性3問）
- **要望**: 4軸×7-8問 = 約30問程度
- **影響**:
  - 診断時間が増加（現在3分→約6-8分）
  - ユーザーが途中で離脱するリスク増加
  - 診断精度は向上する見込み

##### 問題2: ブロックごとのページ遷移
- **要望**: 4軸（4ブロック）ごとにページ遷移
- **現在の実装**: 1問ずつページ遷移（全16問を連続）
- **影響**:
  - URL構造の変更が必要
  - 状態管理の複雑化（ブロック単位での保存）
  - ユーザーに区切りのある進行感を提供

##### 問題3: UIフォーマットの変更（MBTI形式）
- **要望**: 「質問テキスト、尺度説明、6つの尺度による丸いボタン」
- **現在**: 3択のラジオボタン形式
- **影響**:
  - 6段階の Likert 尺度に変更必要
  - 左右にラベル（例: 「そう思う」←→「そう思わない」）
  - 丸いボタンのUIデザイン実装

##### 問題4: デザイントーンの変更
- **要望**: ポップで洗練されたカラフルなデザイン
- **現在**: ダークモード（紫・青基調）
- **影響**:
  - 4軸ごとのカラーテーマ定義が必要
  - 熱量軸：赤/オレンジ系
  - 重心軸：緑/黄緑系
  - 目的軸：青/シアン系
  - 同期軸：紫/ピンク系
  - 全体的に明るくポップな色調に変更

#### 対応方針

1. **質問データの拡張**: 各軸7-8問、計30問程度に増量
2. **ブロックナビゲーション**: 各軸をブロックとして、ブロック内で1問ずつ表示
3. **MBTI形式UI**: 6段階の丸いボタン（円形ラジオボタン）を実装
4. **カラフルデザイン**: 4軸ごとのカラーテーマを定義し、ポップな配色に変更
