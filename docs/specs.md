# 仕様書

## 1. アプリ概要

### RelationCheck 16
16タイプ関係性診断Webアプリケーション

## 2. 4軸スコアリングシステム

### 2.1 熱量軸（Hot / Cold）
| 項目 | 説明 |
|------|------|
| 高スコア側 | 感情的・能動的・熱く語り合える |
| 低スコア側 | 冷静・ドライ・客観的 |
| カラーテーマ | 赤/オレンジ系 (#ff6b6b, #feca57) |
| 質問数 | 7-8問 |

### 2.2 重心軸（Equal / Lean）
| 項目 | 説明 |
|------|------|
| 高スコア側 | 対等な関係・相互に支え合う |
| 低スコア側 | どちらかに偏りがある関係 |
| カラーテーマ | 緑/黄緑系 (#1dd1a1, #10ac84) |
| 質問数 | 7-8問 |

### 2.3 目的軸（Value / Loose）
| 項目 | 説明 |
|------|------|
| 高スコア側 | 成長・生産性重視・目標志向 |
| 低スコア側 | 心地よさ・惰性重視・楽観主義 |
| カラーテーマ | 青/シアン系 (#48dbfb, #0abde3) |
| 質問数 | 7-8問 |

### 2.4 同期軸（Sync / Desync）
| 項目 | 説明 |
|------|------|
| 高スコア側 | 回答内容が一致・価値観が近い |
| 低スコア側 | 回答内容がズレている・価値観が異なる |
| カラーテーマ | 紫/ピンク系 (#ff9ff3, #f368e0) |
| 質問数 | 7-8問 |

## 4. URL構造（階層的ルーティング）

```
/                          ← ホーム（ランディングページ）
├── diagnose/              ← 診断スタート・名前入力
├── diagnose/user1/        ← パートナーAのブロック選択
├── diagnose/user1/block/  ← パートナーAの各ブロック
│   ├── temperature/       ← 熱量軸ブロック
│   ├── balance/           ← 重心軸ブロック
│   ├── purpose/           ← 目的軸ブロック
│   └── sync/              ← 同期軸ブロック
├── diagnose/user2/        ← パートナーBのブロック選択
├── diagnose/user2/block/  ← パートナーBの各ブロック
│   ├── temperature/
│   ├── balance/
│   ├── purpose/
│   └── sync/
├── diagnose/loading/      ← 診断計算中
└── result/                ← 診断結果表示
```

## 5. UI仕様

### 5.1 MBTI形式の質問画面
```
┌─────────────────────────────────────┐
│  ブロック名 - 質問 n / m              │
├─────────────────────────────────────┤
│                                     │
│  【質問テキスト】                     │
│  二人でいる時、感情の起伏は？          │
│                                     │
│  【尺度説明】                         │
│  感情的 ←――――――――――→ 冷静的        │
│                                     │
│    ○  ○  ○  ○  ○  ○             │
│   -3  -1  0  +1  +3                │
│    ↑                               │
│   選択中                            │
│                                     │
│  [前へ]              [次へ/完了]    │
│                                     │
└─────────────────────────────────────┘
```

### 5.2 6段階尺度
- 選択肢: -3, -1, 0, +1, +3（奇数にして中央を0に）
- または: 1, 2, 3, 4, 5, 6（6段階）
- 視覚的に丸いボタンを横一列に配置
- 左右にラベルを表示

### 5.3 カラーテーマ

#### グローバル
- 背景: 白または薄いグレー（#f8f9fa）
- テキスト: 濃いグレー（#2d3436）
- アクセント: 軸ごとのカラー

#### 軸ごとのカラー
| 軸 | メインカラー | サブカラー | グラデーション |
|---|------------|-----------|--------------|
| 熱量軸 | #ff6b6b | #feca57 | linear-gradient(135deg, #ff6b6b, #feca57) |
| 重心軸 | #1dd1a1 | #10ac84 | linear-gradient(135deg, #1dd1a1, #10ac84) |
| 目的軸 | #48dbfb | #0abde3 | linear-gradient(135deg, #48dbfb, #0abde3) |

## 6. 技術仕様

### 6.1 使用技術
- Next.js 14 (App Router)
- React 18
- CSS Modules
- Lucide React（アイコン）

### 6.2 状態管理
- Context APIを使用
- 管理対象:
  - `user1Name`, `user2Name` - ユーザー名
  - `user1Answers`, `user2Answers` - 回答データ（軸ごとに管理）
  - `completedBlocks` - 完了したブロック
  - `result` - 診断結果

### 6.3 診断ロジック
1. 各軸のスコアを平均化（7-8問の回答から）
2. 4軸スコアから16タイプを判定
3. シンクロ率を計算（二人の回答の一致度）
